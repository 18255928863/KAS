Remarks
This event comes at different link states depending on what has initiated the link break: If link has been broken via KAS source (BreakCurrentLink(LinkActorType, Boolean)) then this event is fired before the parts have actually decoupled. If link has been broken externally, e.g. via physics or by invoking decouple(Single) method, then this event is fired after the parts have decoupled.

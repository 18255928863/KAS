KAS Namespace

The main namespace for the KAS functionality implementations.

A third-party code usually don't need to access anything from this namespace. The only legitimate exception is writing a module that inherits a KAS module. In all the other cases a third-party code must use KAS API to interact with the KAS modules.

If inheriting is the case then a depedent mod must include a reference to the KAS.dll assembly. However, this assembly must not be released together with the mod. Having two KAS assemblies in the game may badly break it.

Classes
  ClassDescription
AbstractJointModule
Module that controls a physical joint on a KAS part.
AbstractProceduralModel
Base class for the parts that dynamically create their model on the game load.
BrokenJointListener
Helper class to detect a joint breakage and deliver the event to the part that ctually controls this joint.
KASModuleCableJoint
Module that ties two parts with a cable. It allows any movement that doesn't try to increase the maximum cable length.
KASModuleCableJointBase
Module that controls a physical cable joint on a KAS part.
KASModuleInteractiveLinkSource
Module that allows connecting the parts by a mouse via GUI.
KASModuleKerbalLinkTarget
Module for the kerbal vessel that allows carrying the cable heads.
KASModuleLinkSourceBase
Base link source module. Does all the job on making two parts linked.
KASModuleLinkTargetBase
Base link target module. Only controls target link state.
KASModulePipeRenderer
Module that draws a pipe between two nodes.
KASModulePipeRendererJointConfig
Helper structure to hold the joint model setup.
KASModulePipeRendererModelPipeEndNode
Helper class for drawing a pipe's end.
KASModulePipeRendererWithHead
Extension to the regular render that does't hide the target's joint node. Instead, the node is "parked" at the main part model.
KASModulePipeRendererWithHeadParkedHead
Helper class for drawing a pipe's end.
KASModuleStockJoint
Module that offers normal KAS joint logic basing on the joint created by KSP. The joint is not modified in any way, and its behavior is very similar to the behavior of a regular joint that would normally connect two parts together.
KASModuleTelescopicPipeModel
Module that keeps all pieces of the link in the model. I.e. it's a material representation of the part that can link to another part.
KASModuleTowBarActiveJoint
Flexible link joint designed specifically for towing vessels.
KASModuleTwoEndsSphereJoint
Module that offers a highly configurable setup of three PhysX joints.
KASModuleWinchNew
Module for a simple winch with a deployable connector.

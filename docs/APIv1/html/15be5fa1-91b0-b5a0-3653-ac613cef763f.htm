Parent object.
type
Type: PrimitiveType
Type of the primitive mesh which is the best for wrapping all the meshes of the object. Only Cube, Capsule, and Sphere are supported.
inscribeBoundaryIntoCollider (Optional)
Type: SystemBoolean
If true then collider will define the outer boundaries so what all the meshes are inside the volume. Otherwise, the combined meshes box will define the outer boundary of the collider. It only makes sense for the colliders other than Cube.
Remarks
Intended to create one fast collider at the cost of precision. All the meshes in the parent childs (including the parent) are processed to produce a boundary box. Then, this box is applied to the requested primitive type that defines the shape of the collider.

Note, that rdaius if sphere and capsule is the same in both X and Y axis. If combined boundary box has any of the dimensions significantly different then it makes sense to choose a different collider type. Or break down the hirearchy into more colliders.

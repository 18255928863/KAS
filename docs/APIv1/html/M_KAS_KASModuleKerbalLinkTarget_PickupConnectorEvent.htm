Remarks

The methods that are attributed as KSP events will show up in the part's context menu. Such methods must always be declared as public to let the KSP core correctly handling them. In case of the module class is not sealed, it highly encouraged to define the methods also as virtual.

Avoid accessing the events from the other classes. Instead, the owner class should provide a related fields or methods.

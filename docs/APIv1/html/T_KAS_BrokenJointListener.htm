Remarks
Modules that implement IKasJointEventsListener will get notified about a joint broken on the specific game object. It allows distinguishing joints in a multi-joint setup since normally Unity doesn't tell which joint has broken if there were more than one on the object.
Examples
Let's say a module needs to create an extra joint to something. If it was added to the part then there would be two joints on the game object. When any of them got broken it would be impossible to figure out which one exactly is destroyed. To overcome this limitation an extra game object with a kinematic Rigidbody can be used:

 2  public override void OnStart(StartState state) {
 3    base.OnStart(state);
 4    var jointObj = new GameObject("MyFakeRB");

 6    // Attach new object to the part.

 9    jointObj.transform.localRotation = Quaternion.identity;

12    // Add active modules.

14    jointObj.AddComponent<FixedJoint>().connectedBody = GetTargetBody();
15    jointObj.AddComponent<BrokenJointListener>().hostPart = part;
16  }
17
18  Rigidbody GetTargetBody() {
19    return null;  // Let's pretend it returns a real target object.
20  }

23  public virtual void OnKASJointBreak(GameObject hostObj, float breakForce) {

25    if (hostObj.name == "MyFakeRB") {
